# Shop System Automation 🛒

Автоматизация тестирования системы управления магазином (Web + API) с использованием **Playwright**.

## 🚀 Стек технологий
- **Framework:** [Playwright](https://playwright.dev/)
- **Language:** JavaScript
- **Patterns:** Page Object Model (POM), Helper Libraries

## 📁 Структура проекта
- `tests/` — UI и API тесты (регистрация, логин, админ-панель).
- `tests/pages/` — Page Object классы для UI-тестов.
- `tests/helpers/` — вспомогательные библиотеки:
    - `api.js` — общие методы взаимодействия с API.
    - `constants.js` — конфигурация пользователей и URL.
    - `auth.js` — хелперы для авторизации в UI.
- `playwright.config.js` — конфигурация фреймворка.

## 🛠 Установка
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Anton-Burdey/automation_for_shop.git
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Установите браузеры Playwright:
   ```bash
   npx playwright install
   ```

## 🧪 Запуск тестов
- **Запуск всех тестов (UI + API):**
  ```bash
  npx playwright test
  ```
- **Запуск только API тестов:**
  ```bash
  npx playwright test tests/api.spec.js
  ```
- **Запуск в UI-режиме (Dashboard):**
  ```bash
  npx playwright test --ui
  ```

## 📋 Покрытые сценарии

### 🌐 API Тестирование (TC#1-12)
- **Аутентификация:** регистрация (в т.ч. дубликатов) и логин.
- **Товары:** создание, получение списка, запрос по ID, удаление (CRUD).
- **Корзина:** добавление товаров пользователем.
- **Заказы:** обновление статусов администратором.
- **Negative Testing:** проверка обработки ошибок 401/404/409.

### 🖥 UI Тестирование
- **Регистрация:** валидация полей и создание профиля.
- **Авторизация:** вход под разными ролями.
- **Админ-панель:** управление товарами, складами и статусами заказов.
